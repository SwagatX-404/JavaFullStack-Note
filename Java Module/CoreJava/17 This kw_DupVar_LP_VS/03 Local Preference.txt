/*
 Local Preference Algorithm (LPA)
 ====================================================================
 	1. The LPA is a Compiler's serarching algorithm 
	   for finding the invoked variable declaration statement

	2. This algorithm is named as local preference, because 
	   Compiler first searches the variables declaration statement 
	   in the current method scope, if not found, then only search 
	   in the class scope

	3. As per LPA, Compiler searches 
	   the invoked variable declaration statement as below
	   	1. First in current method body 
		2. Next, in current method parameters
		3. Then next, in the current class scope
		4. Finally, in the current class's super class

	   if comipler can not find the variable's declaration statement 
	   in above all 4 scopes, then it throws error: can not find symbol variable

	4. Compiler never access one method's parameter or local variable
	   in another method. Always it accesses variable either of current method
	   or of current class or of its super class only.
	   
Below program show above proints	   
*/

class Test14_LP {
	static int a = 10;

	
	public static void main(String[] args) {

		System.out.println(a);


	}
}

/*
- In the above program Compiler first searches the 'a' variable declaration 
   in main method body, next searches in main method paranthesis parameters, 
   finally search in class scope
  
- The variable 'a' declaration is found in class scope, 
  then compiler replaces Sopln(a) as Sopln(classname.a), as Test.a
   
- It means compiler links variable to class, to tell to JVM 
  access this variable memory from the class scope directly

- If we access SV directly by its name, it is not direct access
  it is classname.SV
	
		In .java file
			System.out.println(a);
		  is replaced in 

		In .class file as
			System.out.println(Example.a);

*/
