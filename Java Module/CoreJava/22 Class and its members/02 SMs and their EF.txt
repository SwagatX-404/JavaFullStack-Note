Order of execution of SMs
----------------------------
When a class is loaded into JVM,

1) JVM allocates memory to SVs with dvs from TOP to Bottom in the order
     they have declared

2) SVs assignment and SBs logic are executed in the order they are defined
     from TOP to Bottom

3) Finally, main method is called and executed.

4) SMs and SICs are executed only when we invoke explicitly.

Compiler adding changes to SMs in .class file
=================================================
1) Compiler moves all SVs declaration to class level in the order
     they defined from top to bottom

2) Compiler moves all SVs assignments and all SBs logic into single SB
     in the order they defined from top to bottom

3) Finally SB is placed at end of the class.

JVM activities w.r.t SMs
=============================
Then when we run a class with SMs

JVM first allocates memory to all SVs with default values then 
it will go to end of the class, and executes SB, initailzes SVs and executes SBs logic.
